# Asif Health Tracker — Requirements + Wireframe Design Document

## 1. Project Purpose

Build a private, local-first health tracking SPA for Asif to track fat loss, food intake, workouts/walks, cannabis reduction, cannabis inventory, and health progress.

The app should be simple, compact, responsive, and designed for daily use. The main screen should not be overloaded. The first page should always prioritize today’s active logging and overall progress. Historical data, settings, profile, cannabis certification, and inventory should live in separate views.

This document is intended to be pasted into Claude Code or VSCode as the source requirements for building the application.

# 13. Design & Architecture Decisions (Addendum — May 3, 2026)

Resolved during requirements review. These decisions are binding for v1 implementation.

---

## 13.1 Tech Stack

| Decision | Value |
|---|---|
| Framework | React |
| Data storage | Browser localStorage + IndexedDB (no backend) |
| Hosting | Local only (SPA, open index.html or local dev server) |
| "Local hosting settings" in Profile | Toggle to expose on local network (0.0.0.0) |

---

## 13.2 Data Seeding

On first launch, the app must pre-seed:

- Full user profile (Asif, 241.8 lb, all Renpho baseline metrics)
- All 6 cannabis inventory products from Section 5
- Medical cannabis certification from Section 6
- Dietary rules and medical flags

---

## 13.3 Visual Design

| Decision | Value |
|---|---|
| Default theme | Dark mode |
| Tone | Calm, minimal |
| Accent color | Teal / green |
| Density | Compact (maximize data visible per screen) |

---

## 13.4 Today Page — Quick Add

Tapping **Quick Add** opens a picker modal with five options:

1. Food / Meal
2. Cannabis Session
3. Steps
4. Weight
5. Workout / Walk

Each option opens its respective entry modal/drawer.

---

## 13.5 Food Entry UX

- Opens as a modal or bottom drawer
- Name field is free text (no fixed categories)
- Macros entered manually (calories, protein required; carbs, fat optional)
- Meal label is free text with suggestions: Meal, Snack, Munchies Box, Restaurant Meal, Protein Shake, Late Snack, Custom
- User can save any entry as a named template for quick reuse
- Source field: Manual, MyNetDiary Import, Template

---

## 13.6 Cannabis Hard Rule Enforcement

| Rule | Enforcement |
|---|---|
| RSO capsule + infused pre-roll stacking | Hard block — user must explicitly dismiss a blocking confirmation dialog before the session can be logged |
| Cannabis before first meal of the day | Warning toast only — log is allowed |
| Cannabis before workout | Warning toast only — log is allowed |
| Cannabis before grocery shopping | Warning toast only — log is allowed |
| Session count exceeding daily target (>2) | Warning toast only — log is allowed |

---

## 13.7 Medical Clearance

- Lives in Profile under Medical Flags
- Implemented as a toggle switch: "Medical clearance received"
- When toggled on, a confirmation dialog asks for the clearance date
- Confirmed clearance date is stored and shown on the toggle
- Unlocks HIIT, running, heavy lifting, and breathless circuit workout types throughout the app

---

## 13.8 Body Metrics Entry

- **Daily weight**: Logged from the Today page (Weight tile → tap to log today's weight)
- **Full body composition** (body fat %, visceral fat, skeletal muscle, BMR, etc.): Entered in Profile → Body Baseline, updated occasionally (e.g., after a Renpho weigh-in)
- Weight history and body comp history both feed into History charts

---

## 13.9 Steps Tracking

- Manual entry only in v1
- Entered via Quick Add → Steps, or directly on the Steps tile on Today
- Apple Health and Renpho sync deferred to a future version

---

## 13.10 Progress Photos

- Added via browser file picker (no external service)
- Each photo is tagged with:
  - View type: Front / Side / Back
  - Date (defaults to today)
  - Weight at time of photo (manual entry)
- Photos displayed in History (full timeline) and optionally as a small thumbnail on Today

---

## 13.11 Notifications and Alerts

- No browser-level push notifications in v1
- All alerts are in-app only: toast banners and inline warning messages
- Toast alerts are used for cannabis rule warnings, calorie limit approach, step goal nudges

---

## 13.12 Backup and Export

| Format | Purpose |
|---|---|
| JSON (full dump) | Complete app data backup; importable back into the app to restore state |
| CSV per section | Separate files for: food logs, cannabis sessions, workouts, weight history |

Both export options are accessible from Profile → Backup/Export Settings.
